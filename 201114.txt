MVC

애플리케이션을 Model, View, Controller 로 구분해 작업을 분리함으로써,
서로간의 결합성을 최소화하고 유지보수성을 높이며,
개발자들이 각각 맡은 영역에만 집중할 수 있도록 하여 개발의 효율성을 극대화한다.


Model : 서비스와 데이터베이스 처리 처리를 담당하는 역할을 하며,
각 로직 처리, DB질의 처리 기능을 한다.

View : 클라이언트와 서버 간의 인터페이스 역할을 하는 영역으로서
클라이언트로부터 요청받거나 처리된 결과를 보여주는 기능을 한다.

Controller : View 와 Model 을 연결하는 중계 역할을 하며,
클라이언트가 전달한 파라미터를 추출하여 Model 로 전달하고,
처리 결과 보여주는 기능을 한다.


3계층
프레젠테이션 계층 : 뷰 + 컨트롤러
비즈니스 계층 : 실제 요청된 서비스를 처리
영속 계층 : 데이터베이스에 대한 처리
